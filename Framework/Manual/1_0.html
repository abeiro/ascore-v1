2.1 Conceptos

En el desarrollo Web, al igual que en muchas otras tecnologías, gran parte del total de un proyecto es el codigo destinado la gestión de los datos implicados en la apliación. Usualmente partimos de una tabla en BBDD donde almacenamos los datos, y se implementan las operaciones básicas sobre esta tabla: crear registro, editar registro, borrar registro, mostrar registro y listar registros. CoreG2 proporciona un API bien definido para poder disponer de estas operaciones rápidamente.

También es usual en el desarrollo web el uso de varios lenguajes. En el caso de CoreG2 se usa PHP, HTML, JavaScript y SQL. Una mala costumbre presente en muchos proyectos Web es la mezcla de esta amalgama de lenguajes. CoreG2 proprociona una librería de plantillas que usaremos para separa estos lenguajes en función de su uso.

  • PHP: Ejecutado en Servidor. Lógica de la aplicación y sus procesos
  • HTML y JavaScript: Ejecutado en Ciente. Interfaz de Usuario
  • SQL: Ejecutado en Servidor. Se usará sólo en casos especiales y siempre en sitios determinados en la capa de acceso a datos

La separación de estos lenguajes sirve a su vez para separar las diferentes partes de la aplicación, ya que se usan para diferentes objetivos.

Las librerías incluidas en CoreG2 son fruto de un desarrollo a lo largo de años, y cubren diversas necesidades que se presentan en el desarrollo de aplicaciones.
